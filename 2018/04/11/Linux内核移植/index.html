Linux内核移植以及其他环境搭建

发表于2018-04-11|更新于2021-01-12|计算机
|阅读量:

由于之前用的TQ2440还给学长了，所以有重新在OK6410上面重新搭建了一次环境

Uboot移植

配置和编译Uboot源文件

开发板的厂商已经提供了Uboot所以不需要使用网上的Uboot源码来进行移植

  • 首先需要对Uboot进行配置，市面上常见的开发板基本上都有对于的配置文件，所以不需要自己一一去配置 以TQ2440和OK6410为例：
    TQ2440：make TQ2440_config
    OK6410：make forlinx_nand_ram256_config
  • 对配置好的Uboot进行编译 注意要使用交叉编译:
    make ARCH=arm CROSS_COMPILE=arm-linux-

通过以上操作就得到Uboot.bin文件可以烧写到NAND flash 不过需要注意以下几点：
1.在虚拟机中编译Uboot，Uboot的源码不能放在共享目录中（ps：具体原因不知道，可能是因为win和linux之间编码格式之类的不同吧）
2.在编译之前最好make clean 或者 make distclean否则可能会出错

Uboot一些基本命令以及部分环境变量说明

命令 简要说明
printenv/pri 查看当前环境变量的配置
setenv/set 添加、修改、删除环境变量
saveenv/save 保存环境当前环境变量（因为设置之后并没有保存到nand flash中）
tftp 通过网络下载文件
bootm 执行固定格式的二进制程序
环境变量 简要说明
baudrate 波特率
ethaddr MAC地址
gatewayip 网关
ipaddr 本机（开发板）的IP地址
serverip tftp服务器IP（linux虚拟机IP）
netmask 子网掩码（一般是255.255.255.0）
bootdelay uboot启动时的延迟时间
bootcmd 当延时结束时自动执行的命令

ps：使用tftp下载:
 1.设置IP地址（开发板IP）:setenv ipaddr 192.168.2.110
 2设置服务器IP地址（虚拟机IP）：setenv sever ip 192.168.2.120
 3.下载内核：tftp 0xc0008000 uImage
 4.设置自动下载内核：setenv bootcmd tftp 0xc0008000 uImage ; bootm 0xc0008000

Linux内核制作

制作嵌入式linux平台的内核，方法和PC平台的Linux内核基本一致

  • 清楚原有的配置与中间文件
    x86: make distclean
    arm: make distclean

  • 配置内核
    x86：make menuconfig
    arm：make meuuconfig ARCH=arm

  • 编译内核
    x86：make bzImage
    arm：make uImage ARCH=arm CROSS_COMPILE=arm-linux-

    注意以下几点：

1.编译内核的虚拟机最好是32位的，因移植到开发板的是32位的，用64位的需要安装很多支持包
2.如果虚拟机的系统是比较高的版本编译低版本的内核（我用的是ubuntu 16.04 编译3.0的内核）可能会出现下面错误：
Can’t use ‘!defined(@array)’ (Maybe you should just omit the defined()?) at kernel/timeconst.pl
错误提示很明确是 在/kernenl/timeconst.pl文件中将if (!defined(@val)) 改为if (!(@val))，再次编译就可以通过了

3.如果出现
Makefile:416: mixed implicit and normal rules: deprecated syntax
Makefile:1449:mixed implicit and normal rules: deprecated syntax
是因为make工具太新，需要对makefile进行修改
找到提示对应行
config %config: scripts_basic outputmakefile FORCE->%config: scripts_basic outputmakefile FORCE
/ %/: prepare scripts FORCE->%/: prepare scripts FORCE

制作根文件系统

建立跟文件系统目录与文件

  • 创建一个目录即跟文件系统创建的目录，并在里面创建文件系统必须的目录
    mkdir rootfs
    cd rootfs
    mkdir bin dev etc lib proc sbin sys usr mnt tmp var
    mkdir usr/bin usr/lib usr/sbin lib/modules

  • 创建设备文件
    cd rootfs/dev
    mknod -m 666 console c 5 1
    mknod -m 666 null c 1 3
    这2个设备文件是linux内核启动时必须的设备文件

  • 在etc目录中加入配置文件
    这部分的是网上找的所以不做单独说明，直接复制进去即可

  • 添加内核模块
    在之前编译好的目录下编译内核模块，并将内核模块安装到所创建的跟文件系统的目录中
    make modules ARCH=arm CROSS_COMPILE=arm-linux-
    make modules_install ARCH=arm INSTALL_MOD_PATH=”跟文件系统所在的目录“

  • 配置busybox
    busybox的作用：我们创建好的跟文件系统中只有创建好的目录以及之前安装的内核模块和少部分必须的文件，并没有linux一些基本的命令，所以需要busybox为我们的文件系统创建linux命令

1.配置busybox：
 make menuconfig
 Busybox Settings àbuild Options->
  选中 “Build busybox as a static binary”, //静态链接
  Cross Compiler prefix (arm-linux-) //指明交叉工具链
 Installation Options->
  选中 “Don‘t use /usr”, 选中该项可以避免busybox 被安装到宿主系统的/usr目录下，破坏宿主系统
  Busybox Installation Prefix (”跟文件系统的目录“) 该选项表明编译后的busybox的安装位置
2.编译、安装busybox
 make
 make install
注意以下几点：
1.如果出现‘passwd_main….’错误，需要在 include/libbb.h 中添加 #inlcude <resource.h>这个头文件

挂在文件系统到内核

  • 使用Initramfs
    1.配置编译内核
    在跟文件系统中穿件一个软连接 ln –s ./bin/busybox init
    配置Linux内核，支持initramfs
     make menuconfig ARCH=arm
      General setup–>
       Initial RAM filesystem and RAM disk (initramfs/initrd) support //勾选这个选项
        (”跟文件系统的路径“) Initramfs source file(s) (NEW) //并且指明跟文件系统的路径
    配置好后重新编译内核
    2.在Uboot命令行中设置bootargs环境变量
    setenv bootargs noinitrd console=ttySAC0,115200

  • 使用NFS
    1.配置编译内核
    首选取消之前勾选的支持Initramfs的配置选项
    配置Linux内核，支持NFS
    File systems –>Network File Systems(勾选)–>
    勾选Root file system on NFS这个选项

2.在Uboot命令行中设置bootargs环境变量
setenv bootargs noinitrd console=ttySAC0,115200 init=/init root=/dev/nfs rw nfsroot=192.168.2.120（服务器IP）:/home/leungloh/rootfs（跟文件系统的路径）,proto=tcp,nfsvers=3 ip=192.168.2.110（开发板IP）:192.168.2.120（服务器IP）:192.168.2.1（网关）:255.255.255.0::eth0:off
高亮的地方需要根据自己的实际环境进行修改

至此Uboot 内核 跟文件系统全部已经配置完整，烧写到开发板即可

附：

ubuntu 搭建tftp和nfs服务器

  • 搭建tftp服务器

1.安装： apt-get install tftp-hpa tftpd-hpa
2.建立目录：
 mkdir /tftpboot # 这是建立tftp传输目录。
 sudo chmod 0777 /tftpboot
 sudo touch test.txt # test.txt文件最好输入内容以便区分
3.配置：
 vi /etc/default/tftpd-hpa
 TFTP_USERNAME=”tftp”
 TFTP_DIRECTORY=”/tftpboot” # 这里是你的tftpd-hpa的服务目录,这个想建立在哪里都行
 TFTP_ADDRESS=”0.0.0.0:69”
 TFTP_OPTIONS=”-l -c -s” # 这里是选项,-c是可以上传文件的参数，-s是指定tftpd-hpa服务目录，上面已经指定
4.重启服务
  sudo service tftpd-hpa restart
5.测试：
 cd /home
 tftp localhost       #localhost 表示本机
 tftp>get test.txt      //test.txt 是之前在/tftpboot 目录下新建的文件
 tftp>put test1.txt      //test1.txt 是在 /home 目录下新建的文件
 tftp>q
退出后，在/home目录下会有一个test.txt文件，在/tftpboot 目录下有test1.txt，表示tftp服务器安装成功！

  • 搭建NFS服务器

1.安装：sudo apt install nfs-kernel-server
2.配置：
 sudo vi /etc/exports
 /tmp *(rw,sync,no_subtree_check,no_root_squash)

3.创建共享目录：sudo mkdir -p /tmp
4.重启NFS：sudo service nfs-kernel-server restart

在配置中我们的NFS的目录就是跟文件系统所在的目录
