Git常用命令

发表于2020-03-18|更新于2021-01-12|计算机
|阅读量:

总结一下Git常用的几个命令，以及向GitHub提交代码的命令

其实大神stormzhang已经整理过从0开始学习 GITHUB 系列教程，

教你从0开始学习GitHub，相信即使你对GitHub一无所知也能看懂。

Git具体命令

命令 简要说明
git init 初始化创建一个repo
git status 查看当前状态
git add 添加至暂存区
git commit 提交
git branch 查看当前分支，git branch a 创建分支a
git checkout a 切换到分支a
git checkout -b a 新建一个a分支，并且自动切换到a分支
git branch -d 删除分支
git branch -D 强制删除分支
git tag 添加版本标签
git ls-remote 显示远程的HEAD hash值
git ls-files 加入index的文件
git log –graph –pretty=oneline –abbrev-commit 看分支信息用这个比较好
git clean –force -d 清理untracked文件
git merge –no-ff dev 可以保证分支线的独立
git rebase –no-ff dev 这个加上–no-ff参数并没有什么卵用， 等于git merge dev
git revert [commitish] 撤销一个提交，生成一个新的提交，并应用到当前分支
git checkout [file] 恢复暂存区的指定文件到工作区
git checkout [commit] [file] 恢复某个commit的指定文件到暂存区和工作区
git checkout . 恢复暂存区的所有文件到工作区
git reset [file] 重置暂存区的指定文件，与上一次commit保持一致，但工作区不变
git reset –hard 重置暂存区与工作区，与上一次commit保持一致
git reset [commit] 重置当前分支的指针为指定commit，同时重置暂存区，但工作区不变
git reset –hard [commit] 重置当前分支的HEAD为指定commit，同时重置暂存区和工作区，与指定commit一致
git reset –keep [commit] 重置当前HEAD为指定commit，但保持暂存区和工作区不变

向GITHUB 提交代码

命令 简要说明
git push origin master 把本地代码推到远程 master 分支
git pull origin master 远程最新的代码更新到本地
git clone git@github.com:Username/test.gi 把远程test项目 clone 到了本地
git remote add origin git@github.com:Username/test.git /把本地项目与 GitHub 上的 test 项目进行关联
git remote remove origin 取消本地目录下关联的远程库
git remote -v 查看当前项目有哪些远程仓库可以执行
