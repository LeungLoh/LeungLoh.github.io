hadoop家族大数据环境安装部署

前言

网上其实有很多关于hadoop、hbase、hive等环境的安装配置，但是大部分都是某个环境独立的安装配置并且没有写明版本信息，在我看来没写明版本信息的安装都是耍流氓，尤其是大版本之间的变化会导致很多问题，所以根据自己的根据自己的安装版本以及配置做一个总结。

环境配置

软件 版本 配置
system CentOS 7.6 云服务器
jdk 1.8 -
mysql 5.7.30 -
hadoop 2.7.6 伪分布式
hbase 2.1.9 单机
hive 2.3.2 单机

本文主要介绍hadoop相关环境配置，至于jdk、mysql的安装不做过多的赘述，以下按照你hadoop、hive、hbase的安装顺序依次进行安装介绍。

安装部署

设置免密登录

由于后面的操作大部分都需要输入密码，为方便操作使用免密登录，避免重复输入密码，如果想输入密码可以忽略

Hadoop

  • 将安装包下载好后直接解压，并且移动到/usr/local下面

  • 创建后续需要是用到的目录

    • cd /usr/local/hadoop && mkdir -p tmp hdfs hdfs/data hdfs/name

  • 添加环境变量

  • 修改配置文件，配置文件的路径在$HADOOP_CONF_DIR

    • core-site.xml

      • 设置 HDFS NameNode 的 URI 为 IP1:9000
      • 设定了 I/O 文件缓存容量

    • hdfs-site.xml

      • 设定 Namenode 信息存储目录
      • 设定 DataNode 的目录路径
      • 设置副本数为 1
      • 设置 Secondary NameNode URI 为 IP2:9001 (slave1 <=> IP2)
      • 开启 WebHDFS 模块

    • mapred-site.xml(cp mapred-site.xml.template mapred-site.xml)

      • 任务历史默认端口也是 10020
      • 任务历史 Web 界面端口也是 19888

    • yarn-site.xml

  • 启动

  • 验证(需要开放防火墙端口)

    • http://ip:8088/
    • http://ip:50070/
