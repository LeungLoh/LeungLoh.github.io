后台管理项目总结

前言

最初是想要学习前端知识，后来发现内容过于枯燥，且需要记忆的内容太多，所以决定直接找个开源项目做来得实在，在github上找到了一个star比较多的后台管理项目vue-mange-system,且模块比较简单，所以就参照这个开源项目做前端页面，当做完2个页面后感觉需要一个后台来打通连贯性，由于之前一直想学习gin，就趁此机会使用使用gin作为后台框架，来完成这个简单的项目，并以此文章来简单记录学习过程。

前端项目介绍

  • 创建

    原项目使用vue-cli创建，但觉得每次重启特别慢，并且我使用的是vue3所以直接用vite 创建项目的脚手架，使用也非常简单

    使用npm run build用于构建项目，会生成dist的文件目录，将该目录拷贝到服务器上，并使用nginx代理即可

  • 主要模块使用

    • app

      在main.js中创建

    • element-plus

      element plus 是一个开源的组件库，拥有非常丰富的组件，对初学者来说很友好，其实大部分过程都是在熟悉组件的使用

    • router

      在router目录下创建index.js，并在main.js使用app添加到项目中，App.vue中使用<router-view></router-view> 组件嵌入匹配的路由，以下包含了普通路由，子路由，重定向，

    • store

      store 可以理解为全局变量，各个组件都可以访问，但是刷新页面会丢失，如果需要持久保存可以使用localStorage

    • axios

      Axios 是一个基于 promise 的 HTTP 库，可以用在浏览器和 node.js 中,具体可参考官网文档，以下是对axios做了一定封装，可以方便使用

    • 其他：主要是对各种组件的使用，官方文档中包含许多样例可以参考

      为了防止在开发调试的过程中出现跨域的情况，生产环境中使用nginx配置即可，该项目使用cors这个库

    • jwauth

      使用github.com/dgrijalva/jwt-go生成token相关信息，在该项目中使用包含一下几个步骤

      • 生成token，api返回token，通过中间件完成，在每个api的response中都重新生成token并返回，防止token过期
      • 解析请求中携带的token，通过中间件完成，对每个请求（登录除外），解析token是否合法

总结

虽然项目整体来说很简单，而且有很多地方需要去完善，但重要的是期间有所收获，无论是对前端vue的相关知识以及go-gin相关都有了一定了解，希望对后续成长有所帮助。
